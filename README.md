# 🐧 インフラエンジニア向け 実践チートシート＆学習メモ
## 🛠️ コマンドシート（実務で毎日叩くコマンド）

### 1. Linux 必須コマンド
* `pwd` : 自分が今いるディレクトリ（カレントディレクトリ）の絶対パスを表示します。Print Working Directoryの略です。
* `mkdir` : 新しいディレクトリ（フォルダ）を作成します。Make Directoryの略です。
* `cd` : 作成したフォルダの中に移動します。
* `ls -a` : 隠しファイル（.gitなど）も含めて一覧表示するオプションです。
* `cat` : ファイルの中身をすべて画面に出力して確認します。
* `grep` : ファイルの中から特定の文字が含まれる行だけを抽出して表示します。ログ調査の主役です。

### 2. サバイバルVim術
* `i` : 文字を入力するモード（Insertモード）に切り替える。
* `Esc` : 入力モードを終了して、コマンド待ち状態に戻る。
* `:wq` : 保存して終了する（Write & Quit）。
* `:q!` : 保存せずに強制終了する（間違えて設定を消してしまった時などの命綱！）。
* `/` : ファイル内の文字を検索する（`n`で次、`N`で前へ）。
* `cw` : 単語を一瞬で消して入力モードになる（Change Word）。
* `D` : カーソル位置から行末までをバサッと切り捨てる。

### 🚨 トラブルシューティング（現場あるある）
* **`W10: Warning: Changing a readonly file` と言われたら**：
  * **意味**: 「読み取り専用ファイル（編集権限がないファイル）だよ！」という警告。システム設定ファイルなどを一般ユーザーで開いた時によく起きます。
  * **脱出と解決**: 慌てずに `Esc` ＞ `:q!` で強制終了し、`sudo vim ファイル名` で管理者として開き直すのが基本の解決策です。

### 3. Git の基本フロー
1. `git init` : Gitの金庫（.gitフォルダ）を作る。
2. `git add` : 変更をステージング（準備置き場）に上げる。
3. `git commit` : 変更を金庫に永久保存する。
4. `git log` : タイムマシンの履歴を確認する。

## 📚 勉強用学習シート（インフラの裏側・知識メモ）

### 💡 コマンドの覚え方のコツ
* **英単語の略で覚える**：コマンドはすべて、ただの英単語の頭文字です。これを知るだけで定着率が10倍変わります。
* **上矢印（↑）キーを活用**：さきほど打ったコマンドが履歴として呼び出せます。過去の履歴を呼び出して使い回すことで、自然とコマンドの形を指に馴染ませています。

### 🏢 現場環境の違い（Ubuntu vs Red Hat）
* インフラエンジニアの基礎体力の9割は共通スキルで構成されているため、基本のコマンドやVim、Gitの操作はRed Hatでも全く同じように動きます。
* ソフトウェアをインストールする時のコマンドだけは異なります：
  * 練習用（Ubuntu等）: `apt` コマンドを使います（例：`sudo apt install vim`）。
  * 現場用（Red Hat等）: `dnf` （または少し古い環境だと `yum` ）というコマンドを使います（例：`sudo dnf install vim`）。

### 🕰️ タイムマシン（Git）の正体
* 変更履歴が保存された場所は、フォルダの中に隠されている `.git` という秘密のフォルダの中です。
* 最新のファイルが1つあるだけで、過去のすべての記憶はこの `.git` フォルダの中にギュッと圧縮されてしまわれています。
### 4. 権限とシステム管理（セキュリティ・運用）
* `chmod` : ファイルの権限（パーミッション）を変更します。（例: `chmod 400 file.txt` で自分だけが読み取れる強固なロック）
* `sudo su -` : 一時的に最強の特権階級（ルートユーザー）に変身します。用が済んだら必ず `exit` で戻るのが鉄則です。
* `systemctl` : サーバーの裏で動くプログラム（サービス）を管理する守護神です。（例: `systemctl status apache2` で状態確認、`stop` で停止）
* `apt` : （Ubuntu環境での）ソフトウェアのインストールや更新を行うパッケージ管理コマンドです。
### 5. 実際の運用・起動手順（使い方）
* **Webサーバー（Apache）の運用フロー**:
  * 起動する: `sudo systemctl start apache2`
  * 停止する: `sudo systemctl stop apache2`
  * 状態を確認する: `systemctl status apache2`
* **Vimでのファイル編集サバイバルフロー**:
  1. `vim ファイル名` でファイルを開く
  2. `i` を押して入力モード（INSERT）に入る
  3. 編集が終わったら、`Esc` を押してコマンド待ち状態に戻る
  4. `:wq` と入力して `Enter` を押し、保存して終了する
* **ネットワーク疎通確認（インフラの眼）**:
  * `ping -c 4 google.com` : 相手サーバーが生きているか確認する。（通信のボールを4回投げる）
  * `curl -I https://example.com` : Webサーバーからの応答（ステータスコード）だけをスマートに確認する。
* **プロセス監視とトラブル対応（緊急外科手術）**:
  * `top` : システム全体のリソース（CPU/メモリ）をリアルタイム監視する。
  * `ps aux | grep サービス名` : 【プロセス ステータス (process status)】現在動いているプログラム（プロセス）の一覧を表示し、暴走しているプロセスのID（PID）を特定する。
* `kill -9 PID` : 【キル (kill)】指定したプロセスIDのプログラムを問答無用で強制終了する。

### 7. ファイルの圧縮・バックアップ
* `tar -czvf [圧縮ファイル名.tar.gz] [対象ファイル]` : 【テープ アーカイブ (tape archive)】複数のファイルやディレクトリを1つの圧縮ファイルにまとめる。
* `ls -lh` : 【リスト (list) + ヒューマンリーダブル (human-readable)】ファイルサイズを「K(キロ)」「M(メガ)」「G(ギガ)」など、人間が読みやすい単位で表示する。
* `rm *.log` : 【リムーブ (remove)】ワイルドカード(`*`)を使って、特定の条件（例：拡張子が.logのもの）に合うファイルを一括削除する（※本番環境での実行は要注意！）。

### 8. ジョブの自動実行スケジュール（cron）
* `sudo service cron start` : 【クーロン (cron = ギリシャ語の時間に由来)】自動実行の裏方サービス（デーモン）を起動する（※WSL環境などで必須）。
* `crontab -e` : 【クーロン テーブル エディット (cron table edit)】ユーザー専用の自動実行スケジュール帳（テーブル）を開いて編集する。
* `* * * * * [コマンド]` : 左から「分・時・日・月・曜日」を指定するスケジュール設定。すべて `*` の場合は「毎分」自動実行される（環境に左右されないよう、絶対パスで記述するのが鉄則）。

### 9. ユーザー管理（システム管理者業務）
* `sudo useradd -m [ユーザー名]` : 【ユーザー アド (user add)】新しいユーザーを追加する。`-m` オプションで専用のホームディレクトリ（家）も同時に作成する。
* `sudo passwd [ユーザー名]` : 【パスワード (password)】指定したユーザーのログイン用パスワードを設定・変更する。
* `whoami` : 【フー・アム・アイ (Who am I?)】「私は誰？」の通り、現在自分がどのアカウントに変身して操作しているかをポツンと表示する（作業ミス防止の命綱）。

### 10. テキスト処理（ログ分析の魔術）
* `awk '{print $3, $4}' [ファイル名]` : 【オーク (開発者3人の頭文字 Aho / Weinberger / Kernighan)】空白で区切られた特定の列（カラム）のデータだけを縦にすっぽり抽出するログ分析の必須コマンド。
* `sed 's/置換前/置換後/g' [ファイル名]` : 【ストリーム エディタ (stream editor)】ファイル内の指定した文字列を一括で別の文字に置き換えて画面に出力する。

### 11. ファイル転送と同期（運び屋スキル）
* `scp [転送元ファイル] [転送先ディレクトリ/]` : 【セキュア コピー (secure copy)】暗号化された安全な通信を使って、ファイルを別の場所や遠隔のサーバーへコピーする。
* `rsync -av [転送元] [転送先/]` : 【リモート シンクロ (remote synchronization)】変更があった部分（差分）だけを賢く高速に同期する。実務のバックアップの超定番。

### 12. パッケージ管理（ソフトウェアの追加と更新）
* `sudo apt update` : 【アドバンスド パッケージ ツール (advanced package tool) のアップデート】インターネット上の公式倉庫にあるソフトウェアの「最新カタログ（リスト）」をダウンロードして更新する。
* `sudo apt install [パッケージ名]` : 【アプト インストール (apt install)】最新カタログを基に、新しいソフトウェアをインターネットから取得して自動インストールする。
* `tree` : 【ツリー (tree)】ディレクトリの階層構造を、木の枝分かれ（ツリー状）のように分かりやすく視覚化して表示するコマンド。

### 13. セキュリティ管理（SSH鍵認証）
* `ssh-keygen -t ed25519` : 【セキュアシェル キー ジェネレーター (secure shell key generator)】最高クラスの強度を持つ暗号ログイン用の「鍵ペア（秘密鍵と公開鍵）」を作成する。
* `~/.ssh/id_ed25519` : 【アイデンティティ (identity = 秘密鍵)】自分だけの「実印」。絶対に他人に教えてはいけない最も重要な秘密ファイル。
* `~/.ssh/id_ed25519.pub` : 【パブリック (public = 公開鍵)】サーバーに渡す「南京錠」。これをログイン先のサーバーに登録することで、パスワードなしで安全に接続できる。
### 14. 所有者の変更（権限掌握）
* `chown [ユーザー]:[グループ] [ファイル名]` : 【チェンジオーナー (change owner)】ファイルやディレクトリの持ち主（所有者）とグループを変更する。特定の担当者しか触れないようにするセキュリティ管理の要。
